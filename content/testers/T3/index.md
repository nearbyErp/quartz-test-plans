# Тестер 3 — Операционный поток

> [!important] Твоя зона: физика бизнеса
> **Точки · Меню · Столы · Склад · Заказы · Aggregator integrations**  
>   
> Плотный operational flow с RBAC-привязкой к ролям Manager/Cashier. Большая exploratory-часть в Заказах и Агрегаторах.

## Что ты тестируешь

| Область | Что входит |
|---------|------------|
| **Торговые точки (CRUD)** | Создание, редактирование, suspend/resume, soft delete, поиск, фильтры |
| **График работы ТТ** | 7-дневное расписание, выходные, круглосуточно, часовые пояса |
| **POS-терминалы** | CRUD на ТТ, привязка по ФН/РН ККТ, last_seen |
| **Столы (Dine-in)** | CRUD, canvas drag&drop, статусы, назначение официанта |
| **Меню в карточке ТТ** | Отображение, добавление/удаление, фильтрация по прейскуранту |
| **Склад: Приёмки** | Создание, проводка (draft → posted), сверка с ингредиентами |
| **Склад: Списания** | FIFO, проводка, причины |
| **Склад: Остатки** | Просмотр, фильтрация, расчёт stock balance |
| **Заказы (lifecycle)** | new → in_progress → ready → closed/cancelled, оплата, отмена |
| **Возвраты** | Refunds API |
| **X/Z отчёты смены** | Open/close shift, монитор смен, агрегация |
| **Aggregator integrations** | Bindings ТТ ↔ Я.Еда / Маркет Деливери, webhook-приём |

## Workflow дня

> [!tip] Делай в этом порядке
> 1. **[Smoke pass](/03-smoke-pass)** (30 мин) — общий для всех
> 2. **Кейсы по фичам** в порядке P0 → P1 → P2 (см. [Бэклог](#твой-бэклог))
> 3. **Для каждой формы** — прогон [02-form-validation-checklist](/02-form-validation-checklist)
> 4. **RBAC своих разделов** — открой [01-rbac-matrix → §6, 10, 11](/01-rbac-matrix) (ТТ, Склад, Заказы)
> 5. **Exploratory** в Aggregator/Order — фиксируй сценарии и наблюдения, не только баги
> 6. **Перед новым багом** — поиск в [04-known-bugs-index](/04-known-bugs-index)
> 7. **Заводи баг** в Google Docs по [шаблону](/00-templates/bug-report-template)
