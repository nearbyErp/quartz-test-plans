# Тестер 3 — Операции, POS и HR

> [!important] Твоя зона: каждодневная работа кафе
> **POS-кассир (PIN, заказ, оплата, смена) · Меню ТТ · Заказы · Смены X/Z · Расписание · Зарплата · Aggregator**  
>   
> Самая «живая» зона. POS-flow + операционка + HR-цикл — всё что происходит каждый день.

## Что ты тестируешь

### POS (касса)

| Область | Что входит |
|---------|------------|
| **PIN-логин кассира** | Вход на POS, проверка `pos.access`, deactivated-сотрудники |
| **Открытие смены** | Cashier открывает смену на терминале, X/Z-отчёты |
| **Приём заказа** | Выбор товаров из меню, добавление модификаторов, количество |
| **Оплата** | Наличные, карта, RRN, чек |
| **Закрытие заказа** | Перевод в `closed`, печать чека |
| **Отмена заказа** | Отмена в статусе new/in_progress/ready, причина |
| **Закрытие смены** | Z-отчёт, передача наличных |
| **Работа со столами (dine-in)** | Назначение стола, перенос позиций, освобождение |

### Админка операций

| Область | Что входит |
|---------|------------|
| **Меню в карточке ТТ** | Что отображается у клиента: товары + цены + модификаторы + стоп-листы |
| **Заказы — список и детали** | Просмотр в админке, фильтры, статусы |
| **Заказы — отмена админом** | Manager/owner отменяет уже оформленный заказ |
| **X/Z отчёты в админке** | Финансовая сводка смены, топ товаров, CSV-экспорт |
| **Shift Monitor** | Открытые/закрытые смены, статусы «скоро 24h / Z!» |
| **Aggregator integrations** | Bindings ТТ ↔ Я.Еда / Маркет Деливери, webhook-приём |

### HR-цикл

| Область | Что входит |
|---------|------------|
| **Шаблоны смен** | CRUD, лимит 4 на ТТ |
| **Расписание** | Плановые смены сотрудников на месяц вперёд |
| **Time tracking** | Фиксация фактически отработанных часов (Clock in/out отсутствует — feature gap) |
| **Формулы зарплаты** | Hourly/fixed/mixed, по роли / индивидуальные |
| **Ведомости** | Расчёт ЗП за период, статусы (calculated → confirmed → paid), CSV |
| **Активность сотрудников** | Дашборд по часам/начислениям |

## User journey — что проверяешь

Главный сценарий: **обычный рабочий день кафе ИП.**

1. **Утро: Cashier приходит на смену**
   - PIN-логин на POS
   - Открытие смены (X-отчёт пустой)
2. **День: приём заказов**
   - Клиент заказывает кофе → выбор товара → размер «средний» (модификатор) → молоко «соевое» (модификатор) → итого
   - Оплата картой → чек печатается → заказ в `closed`
   - Если зал: заказ привязывается к столу
   - Стоп-лист сработал на товар «X закончился» → товар не виден кассиру
3. **День: ошибки кассира**
   - Открыл заказ — клиент передумал → отмена с причиной
   - Опечатался в количестве — удалить позицию (только в `new`)
4. **Manager в админке**
   - Открывает Заказы → видит сегодняшние, фильтрует по статусу
   - Видит Shift Monitor → его смена «открыта 5ч»
5. **Вечер: Cashier закрывает смену**
   - Z-отчёт (фискальный, день закрыт)
   - Передача наличных по итогу X-отчёта
6. **Месяц: HR-цикл**
   - Manager строит расписание сотрудников на следующий месяц через шаблоны
   - В конце месяца — Time tracking фиксирует фактические часы (если пишут вручную)
   - Считается зарплата по формулам, генерируется ведомость
   - Экспорт CSV для бухгалтерии
