# Тестер 3 — Операционный поток

> **Физика бизнеса.** Точки, склад, заказы — всё что про оперативную работу ТТ. Плотный operational flow с RBAC-привязкой к ролям Manager/Cashier.

## Что ты тестируешь

| Область | Что входит |
|---------|------------|
| **Торговые точки (CRUD)** | Создание, редактирование, suspend/resume, soft delete, поиск, фильтры |
| **График работы ТТ** | 7-дневное расписание, выходные, круглосуточно (отсутствует), часовые пояса |
| **POS-терминалы** | CRUD на ТТ, привязка по ФН/РН ККТ, last_seen |
| **Столы (Dine-in)** | CRUD, canvas drag&drop, статусы free/occupied/reserved, назначение официанта |
| **Меню в карточке ТТ** | Отображение, добавление/удаление, фильтрация по прейскуранту |
| **Склад: Приёмки** | Создание, проводка (draft → posted), сверка с ингредиентами |
| **Склад: Списания** | FIFO, проводка, причины |
| **Склад: Остатки** | Просмотр, фильтрация, расчёт stock balance |
| **Заказы (lifecycle)** | new → in_progress → ready → closed/cancelled, оплата, отмена |
| **Возвраты** | Refunds API |
| **X/Z отчёты смены** | Open/close shift, монитор смен, агрегация |
| **Aggregator integrations** | Bindings ТТ ↔ Яндекс.Еда / Маркет Деливери, webhook-приём |

## Workflow дня

1. **[Smoke pass](../../03-smoke-pass.md)** (30 мин) — общий для всех
2. **Кейсы по фичам** в порядке P0 → P1 → P2 (см. [Бэклог](#твой-бэклог))
3. **Для каждой формы** прогон [02-form-validation-checklist](../../02-form-validation-checklist.md)
4. **Сверка RBAC** для разделов своей зоны — открой [01-rbac-matrix → §6, 10, 11](../../01-rbac-matrix.md) (ТТ, Склад, Заказы)
5. **Exploratory** в Aggregator/Order — фиксируй сценарии и наблюдения, не только баги
6. **Перед заведением бага** — [поиск дублей](../../04-known-bugs-index.md)
7. **Заводи баг** в Google Docs по [шаблону](../../00-templates/bug-report-template.md)

## Особенности зоны

- **POS-flow заморожен по ADR-014** (`erp-pos/mobile/` frozen). На стенде POS-приложение **не основной приоритет**. Сценарии «оплата на POS» — сверять с тест-лидом, что доступно для тестирования
- **Aggregator** — skeleton (M2). Connector Я.Еда ждёт контракт от провайдера. Тестировать что есть: bindings CRUD + webhook stub
- **Меню в карточке ТТ** — известный backend 500 (BUG-060). Пока не починят — exploratory ограничен
- **Многие RBAC-баги в твоей зоне принадлежат T1** (он владелец RBAC matrix), но фактически блокируют тебя:
  - BUG-014 — Manager → ТТ 403
  - BUG-015 — Franchisee → создание ТТ 403
  - BUG-016 — Manager → Склад 403
  - BUG-017 — Manager → Заказы 403
  
  Их прогон делает T1; ты при тестировании просто фиксируешь что воспроизводится в **твоих** разделах
