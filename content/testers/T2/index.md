# Тестер 2 — HR + Каталог

> [!important] Твоя зона: всё про сотрудников после создания + товарный справочник
> **HR-блок** (Шаблоны смен, Расписание, Time tracking, Зарплата, Ведомости, Активность)  
> **+ Весь Каталог** (Товары, Категории, Модификаторы, Прейскуранты, Стоп-листы, Техкарты, Ингредиенты)
>   
> Стабильный домен данных, минимум live-операций — можно копать глубоко.

## Что ты тестируешь

### HR-блок

| Область | Что входит |
|---------|------------|
| **Шаблоны смен** | CRUD, лимит 4 на ТТ |
| **Расписание** | Плановые смены, future-only CRUD, шаблоны → расписание |
| **Time tracking** | Clock in/out (отсутствует — feature gap), ручной ввод, корректировки |
| **Формулы зарплаты** | Hourly/fixed/mixed, по роли / индивидуальные, валидация |
| **Ведомости** | Расчёт, статусы (calculated → confirmed → paid), CSV-экспорт |
| **Активность сотрудников** | Дашборд, агрегаты по часам/начислениям |
| **Терминалы dashboard** | Live через Kafka shiftMonitor — переосмыслить или удалить |

### Каталог

| Область | Что входит |
|---------|------------|
| **Товары** | CRUD, soft delete, КБЖУ, фискальные атрибуты, флаги |
| **Категории** | Дерево, parent_id, активность, каскад деактивации |
| **Модификаторы** | Группы + опции, structural/free, привязка к товарам |
| **Прейскуранты** | CRUD, привязка к ТТ, цены товаров и опций |
| **Стоп-листы** | Per-store блокировка товаров и категорий |
| **Техкарты** | Рецептуры (в Warehouse Service), per-option |
| **Ингредиенты** | Справочник (в Warehouse Service) |

## Workflow дня

> [!tip] Делай в этом порядке
> 1. **[Smoke pass](../../03-smoke-pass.md)** (30 мин) — общий для всех
> 2. **Кейсы по фичам** в порядке P0 → P1 → P2 (см. [Бэклог](#твой-бэклог))
> 3. **Для каждой формы** — прогон [02-form-validation-checklist](../../02-form-validation-checklist.md)  
>    ⚠️ В твоей зоне больше всего форм — это даст массу багов
> 4. **RBAC своих разделов** — открой [01-rbac-matrix → §7-9, 12](../../01-rbac-matrix.md), сверь наблюдения с ожиданием
> 5. **Перед новым багом** — поиск в [04-known-bugs-index](../../04-known-bugs-index.md) на дубли
> 6. **Заводи баг** в Google Docs по [шаблону](../../00-templates/bug-report-template.md)

## Особенности зоны

> [!warning] Versioning каталога — не реализовано
> По ADR-011 решено делать (товар → новая version → catalog draft → published). Сейчас отсутствует (BUG-033, BUG-034). Помечать как gap-feedback, не как regression.

> [!warning] Time tracking — Clock in/out отсутствует
> Только ручной ввод плановой смены. Это **feature gap** (BUG-021), не баг.  
> Сценарии «как сейчас работает ручной ввод» проверять надо.

> [!info] Каталог + PayKeeper
> Фискальные атрибуты товара (`vat_rate`, `payment_subject`, `payment_type`) обязательны для инвойса PK.  
> Без них товар нельзя продать. Проверить валидацию.

> [!note] Что НЕ в твоей зоне
> - Авторизация, Франшизы, Юр.лица, Роли, Сотрудники-CRUD → **[T1](../T1/)**
> - ТТ, Меню в карточке ТТ, столы, склад (приёмки/списания), заказы → **[T3](../T3/)**
