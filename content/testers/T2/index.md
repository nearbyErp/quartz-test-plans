# Тестер 2 — Каталог и склад

> [!important] Твоя зона: ИП заводит ассортимент и сырьё
> **Категории · Товары · Модификаторы · Прейскуранты · Стоп-листы · Техкарты · Ингредиенты · Приёмки · Списания · Остатки**  
>   
> Стабильный домен данных, минимум live-операций — можно копать глубоко. В твоей зоне больше всего форм → больше всего validation-багов.

## Что ты тестируешь

### Каталог (товарная часть)

| Область | Что входит |
|---------|------------|
| **Категории** | Дерево, parent_id, активность, каскад деактивации |
| **Товары** | CRUD, soft delete, восстановление, КБЖУ, фискальные атрибуты, флаги (is_open_price, is_by_weight, is_alcohol и т.д.) |
| **Модификаторы** | Группы + опции, structural/free, привязка к товарам, min/max |
| **Прейскуранты** | CRUD, привязка к ТТ, цены товаров и опций |
| **Стоп-листы** | Per-store блокировка товаров и категорий (для одной ТТ ИП) |

### Склад

| Область | Что входит |
|---------|------------|
| **Ингредиенты** | Справочник (сырьё для приготовления) |
| **Техкарты** | Рецептуры блюд, per-option (для модификаторов) |
| **Приёмки** | Поступление ингредиентов, draft → posted, сверка с поставщиком |
| **Списания** | FIFO, проводка, причины (порча/собственные нужды) |
| **Остатки** | Просмотр текущих запасов, фильтрация |
| **Конвертация единиц** | Per-product/ingredient (например "бутылка" → "мл") |

## User journey ИП-владельца — что проверяешь

Главный сценарий: **ИП заполняет меню кафе и заводит склад.**

1. Создать дерево категорий: «Завтраки» → «Каши», «Омлеты»; «Напитки» → «Кофе», «Чай»
2. Завести товары в каждой категории (5-15 шт). Заполнить КБЖУ, фискальные атрибуты, флаги
3. Создать группы модификаторов («Размер», «Молоко», «Сахар»), привязать к товарам
4. Создать прейскурант, выставить цены товаров и опций модификаторов
5. Привязать прейскурант к своей ТТ
6. Поставить пару товаров в стоп-лист (например, кончился молоко)
7. Завести 10-20 ингредиентов (мука, молоко, кофейные зёрна, фрукты)
8. Создать техкарту блюда: «Капучино = 200 мл молока + 18 г кофе»
9. Сделать приёмку ингредиентов от поставщика
10. Сделать списание (порча, собственные нужды)
11. Посмотреть текущие остатки

## Особенности зоны

> [!warning] Versioning каталога — не реализовано
> По ADR-011 решено делать (товар → новая version → catalog draft → published).  
> Сейчас отсутствует (BUG-033, BUG-034). Помечать как gap-feedback, не как regression.

> [!info] Каталог + PayKeeper / 54-ФЗ
> Фискальные атрибуты товара (`vat_rate`, `payment_subject`, `payment_type`) обязательны для инвойса PK.  
> Без них товар нельзя продать через POS. Проверить валидацию на форме товара.

> [!warning] «Склад не найден для ТТ»
> BUG-066 — при выборе ТТ в Приёмках выкидывает ошибку. Проверь что у ТТ авто-создан склад при создании ТТ. Если нет — это блокер для всей складской зоны.
