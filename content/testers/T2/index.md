# Тестер 2 — HR + Каталог

> **Всё про сотрудников после создания + товарный справочник.** Стабильный домен данных, минимум live-операций — можно копать глубоко.

## Что ты тестируешь

| Область | Что входит |
|---------|------------|
| **Шаблоны смен** | CRUD, лимит 4 на ТТ |
| **Расписание** | Плановые смены, future-only CRUD, шаблоны → расписание |
| **Time tracking** | Clock in/out (отсутствует — feature gap), ручной ввод, корректировки |
| **Формулы зарплаты** | Hourly/fixed/mixed, по роли / индивидуальные, валидация |
| **Ведомости** | Расчёт, статусы (calculated → confirmed → paid), CSV-экспорт |
| **Активность сотрудников** | Дашборд, агрегаты по часам/начислениям |
| **Терминалы dashboard** | Live через Kafka shiftMonitor — переосмыслить или удалить |
| **Каталог: Товары** | CRUD, soft delete, поля КБЖУ, фискальные атрибуты, флаги |
| **Каталог: Категории** | Дерево, parent_id, активность, каскад деактивации |
| **Каталог: Модификаторы** | Группы + опции, structural/free, привязка к товарам |
| **Каталог: Прейскуранты** | CRUD, привязка к ТТ, цены товаров и опций |
| **Каталог: Стоп-листы** | Per-store блокировка товаров и категорий |
| **Каталог: Техкарты** | Рецептуры (в Warehouse Service), per-option |
| **Каталог: Ингредиенты** | Справочник (в Warehouse Service) |

## Workflow дня

1. **[Smoke pass](../../03-smoke-pass.md)** (30 мин) — общий для всех
2. **Кейсы по фичам** в порядке P0 → P1 → P2 (см. [Бэклог](#твой-бэклог))
3. **Для каждой формы** прогон [02-form-validation-checklist](../../02-form-validation-checklist.md)  
   ⚠️ В твоей зоне больше всего форм — это даст массу багов
4. **Сверка RBAC** для разделов своей зоны — открой [01-rbac-matrix → §7-9, 12](../../01-rbac-matrix.md) и сверь свои наблюдения с ожиданием
5. **Перед заведением бага** — [поиск дублей](../../04-known-bugs-index.md)
6. **Заводи баг** в Google Docs по [шаблону](../../00-templates/bug-report-template.md)

## Особенности зоны

- **Versioning каталога** — по ADR-011 решено делать (товар → новая version → catalog draft → published). Сейчас не реализовано (BUG-033, BUG-034). Помечать exploratory-баги по этому в зоне gap-feedback, не как regression
- **Time tracking** — Clock in/out отсутствует (BUG-021). Это feature gap, не баг — но сценарии «как сейчас работает ручной ввод» проверять надо
- **Каталог + PayKeeper** — фискальные атрибуты товара (`vat_rate`, `payment_subject`, `payment_type`) обязательны для инвойса PK. Без них товар нельзя продать. Проверить валидацию
