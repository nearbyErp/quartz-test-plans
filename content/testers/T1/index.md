# Тестер 1 — Setup и персонал

> [!important] Твоя зона: ИП-владелец только запустился
> **Авторизация · Своё ЮЛ · Торговые точки (1-2 шт) · Сотрудники · Роли**  
>   
> 🎯 Главная фича: ты **владелец сквозной [RBAC-матрицы](/01-rbac-matrix)**. В ИП-сценарии активны 3 роли: ИП-владелец (Franchise), Manager, Cashier. Franchisee = N/A.

## Что ты тестируешь

| Область | Что входит |
|---------|------------|
| **Авторизация** | Login, logout, forgot password, reset password, сессии, токены |
| **Профиль ИП / своё ЮЛ** | Редактирование своего главного ЮЛ (название, реквизиты, банк, контакты) |
| **Торговые точки (CRUD)** | Создание 1-2 ТТ, редактирование, suspend/resume, soft delete, поиск, фильтры |
| **График работы ТТ** | 7-дневное расписание, выходные, круглосуточно (отсутствует), часовые пояса |
| **POS-терминалы** | CRUD на ТТ (как объекты), привязка по ФН/РН ККТ, last_seen |
| **Столы (Dine-in)** | CRUD, canvas drag&drop, статусы — если у ИП кафе с залом |
| **Сотрудники** | Найм Manager / Cashier, привязка к 1-2 ТТ, PIN, деактивация |
| **Роли & Permissions** | Системная "Администратор", custom роли, permission catalog |
| **🎯 Сквозная RBAC** | Матрица 3 ролей × 13 разделов — твой основной regression-артефакт |

> [!info] ИП-специфика
> - Раздел «Юр.лица» **скрыт** в меню при `type=individual` — это test case (если стенд `individual`)
> - Создание партнёра (`type=franchisee`) → должно вернуть `403 FRANCHISE_TYPE_INDIVIDUAL`
> - Импорт ЮЛ через xlsx — недоступен для ИП

## Workflow дня

> [!tip] Делай в этом порядке
> 1. **[Smoke pass](/03-smoke-pass)** (30 мин) — если упало, стоп, репорт blocker
> 2. **[RBAC matrix](/01-rbac-matrix)** прогон (~1 час) — 3 учётки × 13 разделов
> 3. **Кейсы по фичам** в порядке P0 → P1 → P2 (см. [Бэклог](#твой-бэклог))
> 4. **Для каждой формы** — прогон [02-form-validation-checklist](/02-form-validation-checklist)
> 5. **Перед новым багом** — поиск в [04-known-bugs-index](/04-known-bugs-index) на дубли
> 6. **Заводи баг** в Google Docs по [шаблону](/00-templates/bug-report-template), присваивай следующий `BUG-NNN`

## User journey ИП-владельца — что проверяешь

Главный сценарий: **ИП открывает кафе и нанимает сотрудников.**

1. Войти под своей ИП-учёткой
2. Открыть свою франшизу, поправить реквизиты главного ЮЛ
3. Создать первую ТТ: адрес, координаты, график работы
4. Прикрутить POS-терминал (фискальник)
5. Если зал — расставить столы на canvas-карте
6. Нанять Manager: создать сотрудника, дать роль с правами на свою ТТ
7. Нанять 2-3 Cashier: создать с PIN, дать роль `pos.access` на свою ТТ
8. Залогиниться под Manager — убедиться что он видит свою ТТ и сотрудников
9. Залогиниться под Cashier — убедиться что в админку не пускает (только POS)

## Учётки нужны

- ИП-владелец (Franchise) — основная для большей части тестирования
- Manager — после создания через систему
- Cashier — после создания через систему
- Если стенд позволяет — отдельная франшиза `type=corporate` для проверки разницы
