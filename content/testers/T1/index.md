# Тестер 1 — Доступы и структура

> **Точка входа в систему + кто кем владеет.** Ты проверяешь что система пускает нужных людей и не пускает ненужных.

## Что ты тестируешь

| Область | Что входит |
|---------|------------|
| **Авторизация** | Login, logout, forgot password, reset password, сессии, токены |
| **Франшизы** | Тип `corporate` vs `individual`, скрытие разделов при `individual`, bootstrap |
| **Юр.лица** | CRUD, импорт xlsx, suspend/resume, владелец партнёра, вкладка «Права» |
| **Роли & Permissions** | CRUD ролей, system "Администратор", hidden roles владельцев партнёров, permission catalog |
| **Сотрудники (CRUD)** | Создание, редактирование, деактивация, реактивация, PIN |
| **🎯 Сквозная RBAC** | Матрица 4 ролей × 13 разделов — твой основной regression-артефакт |

> **Твоя главная фича:** ты владелец [RBAC-матрицы](../../01-rbac-matrix.md). После любого изменения авторизации/permissions — твой прогон выявляет регрессию во всех разделах системы.

## Workflow дня

1. **[Smoke pass](../../03-smoke-pass.md)** (30 мин)  
   Если упало — стоп, репортить blocker
2. **[RBAC matrix](../../01-rbac-matrix.md)** прогон (~1 час)  
   4 учётки последовательно × 13 разделов; расхождения с «Ожидание» = заводишь баг
3. **Кейсы по фичам** в порядке P0 → P1 → P2 (см. [Бэклог](#твой-бэклог))
4. **Для каждой формы** прогон [02-form-validation-checklist](../../02-form-validation-checklist.md)
5. **Перед заведением бага** — поиск в [04-known-bugs-index](../../04-known-bugs-index.md) на дубли
6. **Заводи баг** в Google Docs по [шаблону](../../00-templates/bug-report-template.md), присваивай следующий `BUG-NNN` (координируйся с тест-лидом)

## Учётки нужны

Получить у тест-лида или создать через bootstrap:

- `franchise@test.local` — Franchise owner главной франшизы (`type=corporate`)
- `franchisee@test.local` — Franchisee owner с ≥1 ЮЛ и ≥2 ТТ
- `manager@test.local` — Manager на 1 конкретной ТТ
- `cashier@test.local` — Cashier с PIN на той же ТТ
- Отдельная франшиза `type=individual` для проверки скрытия раздела ЮЛ
