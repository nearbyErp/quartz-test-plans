# ERP Test Plans

Документация ручного тестирования платформы **Obsidian ERP**.

> [!important] Текущий фокус: ИП с 1-2 торговыми точками
> Узкий happy-path сценарий: один владелец, одно главное ЮЛ, 1-2 ТТ, наёмный Manager/Cashier.

## Зоны тестирования

8 функциональных зон. Каждая — со списком функциональных проверок и привязкой известных багов inline. Распределение между тестировщиками — ad-hoc, не вшито в зоны.

1. [Авторизация и доступ](zones/01-auth/) — login, RBAC, permissions
2. [Организация](zones/02-organization/) — Франшиза, ЮЛ, сотрудники, роли
3. [Торговые точки (ТТ)](zones/03-stores/) — ТТ-CRUD, график, POS-терминалы, столы
4. [Каталог](zones/04-catalog/) — категории, товары, модификаторы, прейскуранты, стоп-листы, техкарты
5. [Склад](zones/05-warehouse/) — ингредиенты, приёмки, списания, остатки
6. [HR-цикл (расписание, зарплата)](zones/06-hr/) — шаблоны смен, time tracking, зарплата, ведомости, активность
7. [Заказы и операции (Касса/POS)](zones/07-orders-pos/) — POS-flow, lifecycle заказа, X/Z отчёты, столы dine-in
8. [KDS-flow и интеграции (Кухня)](zones/08-kds-integrations/) — кухонные станции, KDS, связки админка↔KDS↔POS, агрегаторы

## Кто ты?

> [!example] 🧪 Я тестировщик
> Открой свою зону из списка выше. На странице зоны — список функциональных проверок с inline-указанием известных багов (например `[BUG-007]` рядом с пунктом «Поиск по ФИО»).

> [!tip] 🛠 Я разработчик
> Дали баг (например `BUG-013`):
> 1. [Известные баги](04-known-bugs-index.md) — найди по ID контекст, severity, тип
> 2. Если про RBAC → [01-rbac-matrix](01-rbac-matrix.md) расписывает «что должно быть по спеке»
> 3. Бизнес-логика → [dev-документация](https://nearbyerp.github.io/quartz-site/)

> [!note] 📊 Я PM / тест-лид
> - [Известные баги](04-known-bugs-index.md) — сводка ~70 багов, разнесены по зонам
> - [Smoke pass](03-smoke-pass.md) — 30-мин круг для проверки стенда перед циклом
> - [RBAC matrix](01-rbac-matrix.md) — основной regression-артефакт
