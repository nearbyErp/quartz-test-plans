# ERP Test Plans

Документация ручного тестирования платформы **Obsidian ERP**.

> [!info] Этот сайт — навигатор: куда смотреть, что делать, где искать
> Сами баги ведутся в Google Docs (трекер команды). Этот сайт описывает **зоны ответственности**, **процесс**, и индексирует баги для поиска.

---

## Кто ты?

> [!example] 🧪 Я тестировщик
> Открой свою зону:
> 
> - **[Тестер 1 — Доступы и структура](testers/T1/)**  
>   Авторизация, Франшизы, Юр.лица, Роли, Сотрудники-CRUD. **Владелец RBAC-матрицы.** ~17 багов.
> - **[Тестер 2 — HR + Каталог](testers/T2/)**  
>   Шаблоны смен, Расписание, Зарплата, Активность + весь Каталог (товары/категории/модификаторы/прейскуранты/стоп-листы/техкарты). ~22 бага.
> - **[Тестер 3 — Операционный поток](testers/T3/)**  
>   Торговые точки, Меню ТТ, Столы, Склад, Заказы, Aggregator integrations. ~26 багов.
> 
> Не знаешь свою зону — спроси тест-лида.

> [!tip] 🛠 Я разработчик
> Дали баг (например `BUG-013`) — что делать:
> 
> 1. [Известные баги](04-known-bugs-index.md) — найди по ID контекст, severity, тип
> 2. Если про RBAC → [RBAC matrix](01-rbac-matrix.md) расписывает «что должно быть по спеке»
> 3. Бизнес-логика и архитектура → [dev-документация](https://nearbyerp.github.io/quartz-site/)

> [!note] 📊 Я PM / тест-лид
> - [Известные баги](04-known-bugs-index.md) — сводка 70 багов: T1/T2/T3, severity, типы
> - [Smoke pass](03-smoke-pass.md) — 30-мин круг для проверки стенда перед циклом
> - [RBAC matrix](01-rbac-matrix.md) — основной regression-артефакт

## Общие артефакты (используют все)

| Артефакт | Когда нужен |
|----------|-------------|
| [01 RBAC Matrix](01-rbac-matrix.md) | Каждый regression-цикл. Владелец — T1, используют все |
| [02 Form Validation Checklist](02-form-validation-checklist.md) | Применяется к **каждой форме** во всех зонах |
| [03 Smoke Pass](03-smoke-pass.md) | **Первое действие** в каждом цикле. 30 мин. Если упало — стоп |

## Соглашения по ID

| Сущность | Формат | Пример |
|----------|--------|--------|
| Test case | `TC-{T#}-{AREA}-{NNN}` | `TC-T1-AUTH-001` |
| Bug | `BUG-{NNN}` | `BUG-042` |
| Test run | `RUN-{YYYY-MM-DD}-{N}` | `RUN-2026-05-04-1` |
